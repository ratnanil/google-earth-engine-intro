Geospatial Analysis in the Cloud

Why in the Cloud?

Data Explosion
  • We in the middle of an explosion of Earth Observation data
  • Growth of Data is outpacing the growth of desktop harddrives AND internet bandwidth capacity

Paradigm Shift

  • Before: Buy CD's, put them into your cd player
  • Now: Store the data on servers, stream only what you need
Which EO Platform?

  • Google Earth Engine (GEE)
  • Sentinel Hub
  • Open Data Cube (ODC)

  • JEODPP
  • pipsCloud
  • ...
  • openEO
Over the years, a large number of EO-Platforms have emerged. I find the openEO plattform to be especially promising

openEO develops an open API to connect R, Python,
JavaScript and other clients to big Earth observation
cloud back-ends in a simple and unified way.

OpenEO:
  • is an open source project
  • provides a standardized API (is a wrapper around existing plattforms)
  • this adds a layer of additional complexity
  • for, is best accessed via the R API which obfuscates the "cloud" aspect
Carbon footprint?

cloudcarbonfootprint.org
  • Cloud computing is a major contributor to global warming
  • There are a number of tools to calculate the carbon footprint of your cloud usage
  • It is important to be aware of this

Arrays

Dictionaries 

								// Curly brackets are used to define "dictionaries"
								var my_dict = {'food':'bread', 'color':'red', 'number':42};

								// Square brackets are used to access items (by key).

								// Or you can use the dot notation to get the same result.
								my_dict.color;
